# Vision-RCP — Agent Adapter Implementation Design

---

## 🧠 Overview

The **Agent Adapter Layer** is the core abstraction that enables Vision-RCP to control different local AI agents (Claude Code, Codex, Antigravity, etc.) through a unified interface.

> It acts as a translation and control layer between Vision-RCP and agent runtimes.

---

## 🎯 Responsibilities

An Agent Adapter must:

* Accept chat messages from remote UI
* Forward them to the local agent runtime
* Stream agent responses back to Vision-RCP
* Handle interrupts and lifecycle management
* Emit structured events (logs, approvals, status)

---

## 🧩 High-Level Architecture

```text
Remote UI (Chat)
        ↓
Vision-RCP Core (Daemon)
        ↓
Agent Adapter
        ↓
Local Agent Runtime
```

---

## 🧱 Base Adapter Interface

All adapters must implement this contract:

```python
from abc import ABC, abstractmethod

class AgentAdapter(ABC):

    @abstractmethod
    def start(self, session_config):
        """Initialize agent in project context"""
        pass

    @abstractmethod
    def send_message(self, message: str):
        """Send user message to agent"""
        pass

    @abstractmethod
    def stream_output(self):
        """Yield streaming output from agent"""
        pass

    @abstractmethod
    def interrupt(self):
        """Interrupt execution"""
        pass

    @abstractmethod
    def stop(self):
        """Stop agent session"""
        pass
```

# 🔹 Claude Adapter (Interactive CLI)

## Strategy

* Spawn persistent subprocess
* Maintain stdin/stdout streams
* Treat as continuous chat session

---

## Implementation

```python
import subprocess

class ClaudeAdapter(AgentAdapter):

    def start(self, config):
        self.process = subprocess.Popen(
            ["claude"],
            stdin=subprocess.PIPE,
            stdout=subprocess.PIPE,
            stderr=subprocess.STDOUT,
            text=True,
            cwd=config["project_path"]
        )

    def send_message(self, message):
        self.process.stdin.write(message + "\n")
        self.process.stdin.flush()

    def stream_output(self):
        for line in self.process.stdout:
            yield {
                "type": "AGENT_MESSAGE",
                "content": line.strip()
            }

    def interrupt(self):
        self.process.send_signal(subprocess.signal.SIGINT)

    def stop(self):
        self.process.terminate()
```

---

## Challenges

* Output parsing ambiguity
* Streaming boundaries unclear
* Requires buffering strategies

# 🔹 Antigravity Adapter (Script-Based)

## Strategy

* Map message → task
* Spawn script
* Parse logs and detect structured signals

---

## Implementation

```python
class AntigravityAdapter(AgentAdapter):

    def start(self, config):
        self.cwd = config["project_path"]

    def send_message(self, message):
        task_map = {
            "scout": "market_scout.py",
            "signals": "signal_engine.py",
            "risk": "risk_manager.py"
        }

        task = self._infer_task(message)

        self.process = subprocess.Popen(
            ["python", task_map[task]],
            stdout=subprocess.PIPE,
            stderr=subprocess.PIPE,
            text=True,
            cwd=self.cwd
        )

    def _infer_task(self, message):
        if "scout" in message:
            return "scout"
        if "signal" in message:
            return "signals"
        return "risk"

    def stream_output(self):
        for line in self.process.stdout:
            yield self._parse_line(line)

    def _parse_line(self, line):
        if "[APPROVAL]" in line:
            return {
                "type": "APPROVAL_REQUEST",
                "message": line.strip()
            }

        return {
            "type": "AGENT_MESSAGE",
            "content": line.strip()
        }

    def interrupt(self):
        self.process.terminate()

    def stop(self):
        pass
```

# 🧠 Adapter Registry

## Purpose

* Dynamically register adapters
* Enable pluggable architecture

---

## Implementation

```python
class AdapterRegistry:

    def __init__(self):
        self.adapters = {}

    def register(self, name, adapter):
        self.adapters[name] = adapter

    def get(self, name):
        return self.adapters[name]()
```

---

## Usage

```python
adapter = registry.get("claude")
adapter.start(session_config)
```

# ⚠️ Anti-Patterns to Avoid

* ❌ Hardcoding agent logic in daemon
* ❌ Treating all agents as terminal processes
* ❌ Parsing logs blindly without structure
